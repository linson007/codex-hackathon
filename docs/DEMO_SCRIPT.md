```bash

```

If `npm link` fails with permissions under `/usr/local/lib/node_modules`, continue using `./bin/contextos.js` or configure npm to use a user-owned global prefix. Avoid `sudo npm link` unless you intentionally want root-owned global npm files.

## Create Or Rebuild The Knowledge Base

```bash
rm -rf ~/.contextos/kbs/retail-platform

./bin/contextos.js init retail-platform
./bin/contextos.js repos add retail-platform samples/retail-platform/order-service
./bin/contextos.js repos add retail-platform samples/retail-platform/billing-service
./bin/contextos.js repos add retail-platform samples/retail-platform/notification-service
./bin/contextos.js update retail-platform
./bin/contextos.js docs generate retail-platform --force
```

`--force` regenerates repository, service, and endpoint docs even when cached docs already exist. If `OPENAI_API_KEY` is set, the LLM-written onboarding docs are generated and shown by default in the UI. If the key is missing, ContextOS still generates deterministic graph-backed docs.

By default, KBs are stored under `~/.contextos/kbs`. To use a different location:

```bash
export CONTEXTOS_HOME="/path/to/contextos-data"
```

List available knowledge bases:

```bash
./bin/contextos.js kbs
```

Expected columns include knowledge base name, repo count, node count, edge count, created time, and last updated time.

Remove a knowledge base:

```bash
./bin/contextos.js kbs remove retail-platform
```

List repositories in the KB:

```bash
./bin/contextos.js repos list retail-platform
```

## CLI Demo

```bash
./bin/contextos.js ask retail-platform "What is impacted if I change refund eligibility logic?"
```

Use generated onboarding docs as additional explanatory context:

```bash
./bin/contextos.js ask retail-platform "What is impacted if I change refund eligibility logic?" --with-docs
```

The default ask flow uses graph facts only. `--with-docs` sends the graph facts plus cached repository/service/endpoint docs to the answer synthesizer.

View generated repository onboarding docs from the CLI:

```bash
./bin/contextos.js docs view retail-platform order-service
./bin/contextos.js docs view retail-platform order-service --variant deterministic
```

Optional: export tool integration JSON for Jira, Backstage, CI, or custom automation:

```bash
./bin/contextos.js export retail-platform --format json
```

Show Jira planning context without creating an issue:

```bash
./bin/contextos.js jira-plan retail-platform --ticket "Change refund eligibility logic"
./bin/contextos.js jira-plan retail-platform --ticket "Change refund eligibility logic" --project CTX --json
```

Create a real Jira Cloud issue after setting `JIRA_BASE_URL`, `JIRA_EMAIL`, `JIRA_API_TOKEN`, and `JIRA_PROJECT_KEY`:

```bash
./bin/contextos.js jira-plan retail-platform --ticket "Change refund eligibility logic" --create
```

Service and endpoint docs are generated by the same command and are available in the UI under Services & Endpoints. The UI defaults to the LLM variant when available and lets you toggle to deterministic facts.

Expected themes:

- `order-service` owns refund eligibility.
- `billing-service` is called for refund preview/payment.
- `notification-service` consumes `refund-events`.
- Tables include `orders`, `refund_decisions`, `payments`, and `refunds`.
- Suggested files include controller, service, entity, repository, migration, and listener classes.

## Optional OpenAI Setup

Without an API key, ContextOS returns a deterministic graph-backed fallback answer.

To enable OpenAI answer synthesis and LLM-generated onboarding docs:

```bash
export OPENAI_API_KEY="your_api_key_here"
export OPENAI_MODEL="gpt-5.2"
```

Then regenerate docs and run the ask command again:

```bash
./bin/contextos.js docs generate retail-platform --force
./bin/contextos.js ask retail-platform "What is impacted if I change refund eligibility logic?"
./bin/contextos.js ask retail-platform "What is impacted if I change refund eligibility logic?" --with-docs
```

## UI Demo

Start both the API and UI with one command:

```bash
./bin/contextos.js ui
```

This serves all knowledge bases from `~/.contextos/kbs`, not just one selected KB.

Optional ports:

```bash
./bin/contextos.js ui --port 4317 --ui-port 5173
```

Open `http://localhost:5173`, choose a knowledge base, and ask the same refund eligibility question.

For new-joiner onboarding:

- The Ask panel streams OpenAI answer text into the UI when `OPENAI_API_KEY` is available.
- Without an API key, the same panel shows the deterministic graph fallback result.
- Use the suggested question chips under the Ask input for quick demo flows.
- Use the `Export JSON` header button only if you want to show the optional tool payload for Jira, Backstage, CI, or custom automation.
- Scroll to Repository Documentation, click a repo card, and review the repo overview.
- Scroll to Services & Endpoints, click a service or API, and review its focused doc and sequence diagram.
- Use the Dependency Graph panel to pan and zoom the React Flow graph.
- Toggle dark mode from the header to show the persisted theme support.
- Use Generate/Regenerate if docs are missing or stale.

## Updating After Code Changes

If source code changes in any registered repo, refresh the graph:

```bash
./bin/contextos.js update retail-platform
./bin/contextos.js docs generate retail-platform --force
```

Then refresh the browser or ask again from the CLI.

## Stop Running Servers

Stop API and UI from the CLI:

```bash
./bin/contextos.js stopui
```

Equivalent:

```bash
./bin/contextos.js ui --stop
```

You can also press `Ctrl+C` in the terminal running `./bin/contextos.js ui`.

If a port is stuck:

```bash
lsof -ti :4317 | xargs kill -9
lsof -ti :5173 | xargs kill -9
```

## Skill Demo

Show the skill file:

```bash
open skills/contextos-codebase-analysis/SKILL.md
```

Demo prompt for Codex or Claude:

```text
Use ContextOS to understand what files I should inspect before changing the owner creation flow in spring-petclinic.
```

Expected agent commands:

```bash
./bin/contextos.js kbs
./bin/contextos.js update spring-petclinic
./bin/contextos.js ask spring-petclinic "What files should I inspect before changing owner creation flow?" --with-docs --verbose
```

The key point: the agent is not guessing from prompt context. It is using the local ContextOS KB to retrieve graph facts, docs, impacted files, endpoints, tables, and dependencies before answering or editing.
