Problem

Enterprise codebases are hard to understand quickly.

New joiners and AI agents both struggle with scattered repository knowledge.

Slow onboarding

APIs, business flows, tables, topics, and owners are spread across repos.

Risky changes

A small logic change can affect services, endpoints, events, and data paths.

AI context limit

Agents cannot keep an entire multi-repo product in the prompt window.

Stale docs

Architecture notes drift unless they are regenerated from source facts.

Why it matters2 / 10
Solution

A local knowledge graph built from source code.

ContextOS scans repositories, stores graph facts, generates onboarding docs, and answers questions with grounded evidence.

1
Register local repositories in a knowledge base.
2
Scan controllers, services, clients, tables, topics, and files.
3
Generate docs and answer questions from graph evidence.
4
Expose the same context to humans in UI and agents through SKILL.md.
Core idea3 / 10
Demo Scenario

Refund eligibility impact analysis.

One question should find impacted services, APIs, tables, Kafka topics, and source files.

./bin/contextos.js ask retail-platform \ "What is impacted if I change refund eligibility logic?" \ --with-docs Expected: - order-service owns eligibility - billing-service handles refund payment - notification-service consumes refund-events - tables, endpoints, risks, and files listed

Architecture

Facts first, synthesis second.

OpenAI improves wording and reasoning, but the source of truth is the deterministic graph.

A
Local repos scanned into SQLite KB.
B
Graph retrieval selects relevant nodes, edges, files, and docs.
C
OpenAI receives compact evidence only, not whole repositories.
D
Fallback answers work even without an API key.

AI Agent Workflow

# Agent uses the skill contextos kbs contextos update retail-platform contextos ask retail-platform \ "What files should I inspect for refund eligibility?" \ --with-docs # Result becomes grounded task context.
Agent value8 / 10
Differentiators

Local, grounded, and useful without platform setup.

  • Local-first: code and KBs stay on the developer machine.
  • Graph facts are deterministic and repeatable.
  • LLM calls are optional and grounded by evidence.
  • Same KB serves human onboarding and AI agent context.
  • Works with local repositories before enterprise integrations.
  • JSON/Jira/Backstage exports are optional tool paths, not the core dependency.
